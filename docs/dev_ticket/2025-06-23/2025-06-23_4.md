## 技術詳細

### 対象システム・モジュール

**バックエンド**:
- `internal/domain/service/line_service.go` - LINE API連携
- `internal/interfaces/handler/line_simple_handler.go` - LINE認証ハンドラー
- `internal/domain/entity/user.go` - ユーザーエンティティ拡張
- `internal/infrastructure/migration` - DB マイグレーション

**フロントエンド**:
- LINEログインボタンコンポーネント
- 認証状態管理（Context/Store）

### 実装方針

1. **段階的実装**: まず基本的なLINE認証フローを実装
2. **セキュリティ対策**: CSRF対策（state parameter）
3. **エラーハンドリング**: 各段階での適切なエラー処理
4. **ユーザー体験**: スムーズなログインフロー

### 技術スタック

- **バックエンド**: Go, Gin, PostgreSQL
- **フロントエンド**: Next.js, TypeScript, Tailwind CSS
- **認証**: LINE Login API v2.1
- **データベース**: PostgreSQL（USER テーブル拡張）

### 考慮事項

- **パフォーマンス**: LINE API呼び出しの効率化
- **セキュリティ**: アクセストークンの適切な管理
- **互換性**: 既存ユーザー管理システムとの互換性
- **スケーラビリティ**: 将来的な他のソーシャルログイン対応

## API・DB変更

### API変更

**新規エンドポイント**:
- `GET /api/v1/auth/line/url` - LINE認証URL取得
- `GET /api/v1/auth/line/callback` - LINEコールバック処理（GET）
- `POST /api/v1/auth/line/callback` - LINEコールバック処理（POST）

### DB変更
