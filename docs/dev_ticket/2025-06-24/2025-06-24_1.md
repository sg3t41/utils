## 実装内容

### 1. 共通エラーハンドリング関数の拡張 (`common.go`)

```go
// HandleUseCaseError - ユースケースエラーの統一処理
func HandleUseCaseError(c *gin.Context, err error, resourceName string)

// HandleAuthError - 認証関連エラーの処理
func HandleAuthError(c *gin.Context, err error)

// SendErrorWithCodeResponse - エラーコード付きエラーレスポンス
func SendErrorWithCodeResponse(c *gin.Context, statusCode int, message string, code string)

// ValidateUserPermission - ユーザー権限の検証
func ValidateUserPermission(c *gin.Context, targetUserID string) bool

// ParseOptionalStringParam - オプショナルな文字列パラメータの解析
func ParseOptionalStringParam(c *gin.Context, key string) *string
```

### 2. ArticleHandlerのリファクタリング

- 全エラーハンドリングを`HandleUseCaseError`に統一
- オプショナル文字列処理を`processOptionalString`ヘルパーで統一

### 3. UserHandlerのリファクタリング 

- GetUsers関数を以下に分割：
  - `buildGetUsersInput` - 入力データ構築
  - `sendUsersResponse` - レスポンス送信
  - `sendCursorPaginatedResponse` - カーソルページネーション処理
- 権限チェックを`validateUserAccess`に統一
- エラーハンドリングを`HandleUseCaseError`に統一

### 4. AuthHandlerのリファクタリング

- `ErrorResponse`構造体を削除（`StandardErrorResponse`を使用）
- 全認証エラーを`HandleAuthError`に統一
- 他のエラーを`SendErrorWithCodeResponse`に統一
