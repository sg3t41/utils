## 開発日誌を作成すること

何かを実装したり機能を変更した場合、その都度必ず日誌を書いてください。
`docs/dev_diary/yyyy-mm-dd/{チケット番号}.md`の形式で開発日誌を作成してください。内容は以下の通りです。

- **日付**: yyyy-mm-dd hh:mm
- **対応チケット**: `#{チケット番号}`
- **作業内容**:
  - 何をしたか
  - どのような問題が発生したか
  - どのように解決したか
  - 実装したコードの場所・ファイル名
- **テスト結果**:
  - 実行したテストの種類と結果
  - 発見したバグや課題
- **次回の予定(さらなる改善点など)**:
- **学習・知見**: 新しく学んだことや技術的な発見
- **愚痴・雑感**: 思いのままに書いてみて

### 開発日誌のファイル名例

- チケット `2025-06-23_1` の開発日誌 → `docs/dev_diary/2025-06-23/2025-06-23_1.md`
- チケット `2025-06-23_2` の開発日誌 → `docs/dev_diary/2025-06-23/2025-06-23_2.md`
- チケット `2025-06-24_1` の開発日誌 → `docs/dev_diary/2025-06-24/2025-06-24_1.md`

### 複数回作業する場合

同一チケットで複数回作業する場合は、同じファイルに追記してください。日付ごとにセクションを分けて記録することを推奨します。

```markdown
# 開発日誌 - #2025-06-23_1

## 2025-06-23 10:00

**作業内容**:

- 初期実装を開始
  ...

## 2025-06-23 15:30

**作業内容**:

- バグ修正
  ...
```

## 開発ロードマップをチケットに適宜分けるよう心がけて

規模の大きな開発計画ではタスクをチケットに分割してください。
各チケットは並行に開発を進めることを想定しても問題のない範囲で切ってください。

### チケット管理ルール

- **作成**: `docs/dev_ticket/yyyy-mm-dd/yyyy-mm-dd_{n}.md` ({n}は連番)
- **進行中**: `docs/dev_ticket/yyyy-mm-dd/in_progress/` に移動
- **完了**: `docs/dev_ticket/yyyy-mm-dd/done/` に移動
- **保留**: `docs/dev_ticket/yyyy-mm-dd/pending/` に移動
  チケット番号は日付+連番で統一し、開発日誌にも必ず記載すること。

## 開発環境とコマンド

### プロジェクト構造

このプロジェクトは**Go (Gin) + Next.js + PostgreSQL**で構築された個人用ダッシュボードアプリケーションです。

**Clean Architecture / DDD**に基づいた設計:
- `api/cmd/`: アプリケーションエントリポイント
- `api/internal/domain/`: エンティティ、リポジトリインターフェース、ドメインサービス
- `api/internal/application/`: ユースケース層
- `api/internal/infrastructure/`: 外部サービス実装、永続化層
- `api/internal/interfaces/`: HTTPハンドラ、ルーター、DTO
- `frontend/`: Next.jsフロントエンド

### 開発コマンド

**全体起動（推奨）**:
```bash
docker-compose up --build
```

**個別起動**:
```bash
# バックエンド
cd api && go run cmd/server/main.go

# フロントエンド  
cd frontend && npm run dev
```

**データベースマイグレーション**:
```bash
make migrate-up     # マイグレーション実行
make migrate-down   # ロールバック
make migrate-status # ステータス確認
```

**テスト実行**:
```bash
# Go単体テスト
cd api && go test ./...

# LINE Bot Webhookテスト
./test_linebot.sh
```

**開発URL**:
- フロントエンド: http://localhost:3000
- API: http://localhost:8080
- データベース: localhost:5432
